# Alfred SVG 表情调试指南

本指南介绍了如何在本地预览和修改项目中的机器人智能表情 (`.svg`) 文件。由于固件渲染具有一定的环境特性（比如硬件屏幕纯黑底色以及不具备矢量软件预览那样宽容度高的渲染环境），本地网页调试将是您修改这些表情时的最佳方式。

## 目录结构

在 `scripts/alfred_svg/` 目录下，您会看到如下关键文件：
- `*.svg`: 各种机器人的状态和表情（如 `angry.svg`, `thinking.svg` 等）。
- `generate_preview.py`: 用于帮您将所有的 SVG 代码提取、排版并统一生成网页版大预览的辅助 Python 脚本。
- `preview.html`: 生成好的网页文件，即您需要打开用来直接看表情最终成品的渲染网页。

## 如何预览和调试？

如果您在此目录下对原有的 `.svg` 文件进行了任何代码修改（比如更改了线条粗细、眼睛大小、嘴巴弧度等），或是新建了表情文件想要查看效果，只需以下两步操作：

### 1. 更新生成预览网页

在终端中定位到该 SVG 目录 (`scripts/alfred_svg`)，然后执行：

```bash
python3 generate_preview.py
```

这个脚本会自动扫描目录下所有的 `.svg` 文件，将每一段 SVG 代码以嵌入内联（Inline SVG）的方式融合到网页框架中，并赋予它们类似真实机器黑屏设备大小（320 x 240 像素尺寸比例）的矩形容器。这样避免了用系统照片预览器或常见 IDE 打不开源代码、缺少底色看不清绿色线条等各种情况。

### 2. 在浏览器中打开并刷新查看

完成运行后（不到 1 秒即可），您可以立即使用任意常用的浏览器（例如 Google Chrome 或者 Safari）打开或重新刷新目录中已经存在的 `preview.html` 文件：

```bash
open preview.html
```

在打开的页面中，您就可以像看“全家福图册”一样，直观地审查各个表情的视觉中心点、线条粗细在整体画框中是否协调等。

## 注意事项与最佳实践

- **统一的设备色彩体系 (`#39ff14`)**: 目前全系列表情统一使用的是亮绿色作为在暗底硬件屏幕上的高对比度表现。在绘制和修改代码时请尽量保持其主色调和线条风格统一。
- **弃用系统级纯文本和字体 (`<text>`)**: 不要使用 `<text>` 标签在 SVG 中直接写字（如在 SVG 里面直接放置 "Zzz" 或 "Hi"），因为嵌入式的真实硬件设备往往由于缺少完整的字体库而无法渲染。请务必使用纯粹的线段 (`<path>`, `<polyline>`, `<line>`) 和基础几何形状（如 `<circle>`, `<rect>`）将这些字符或表情重新“画出来”。
- **基础参考轴板 ViewBox**: 我们现在默认采用了 `128x128` 的统一视区规范框 `<svg viewBox="0 0 128 128">`，但在硬件 `320x240` 等宽屏上的实际渲染逻辑中，它们会被整体做居中对齐排版。所以在添加图形时，尽量保证重点表现的内容处于该 `viewBox` 画布的主视觉中央。
